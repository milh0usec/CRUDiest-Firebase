AngularFire CRUD app with AngularJS, Firebase, Bootstrap, and async typeahead.

This is a front-end app. Instead of a back-end it's accessing Firebase, a NoSQL cloud database. There are two views: INDEX/NEW (Home) and SHOW/EDIT. I also made a MEAN stack version of this app. The MEAN stack app lacks auth, has seperate SHOW and EDIT pages, and the EDIT pages has a "Submit" button. In contrast, the Firebase SHOW/EDIT page instantly saves user edits to the remote database without the need for a "Submit" button.

Auth is handled by Firebase. The Google, Facebook, Twitter, and GitHub logins are OAuth2. The e-mail & password login opens a UI Bootstrap modal window, with UI Bootstrap alerts and UI Bootstrap tabs. The e-mail & password login has a full feature set, including create new user, check for existing e-mail address, login, change e-mail address, change password, and reset lost password. Lastly, users can login anonymously.

Upon login, a "Add a Movie" data entry form shows. This form uses the UI Bootstrap asynchronous typeahead. Each keystroke makes an HTTP request to the Open Movie Database, which returns ten movie titles. onSelect (clicking a movie title) fires an HTTP GET request for a single movie to the OMDB which returns the movie data. This movie is then pushed locally to the $scope.movies array. This updates the INDEX view almost instantly. Then, running in the background, an HTTP POST adds the movie to the CRUDiest Movies Database. Lastly, an HTTP GET request to the CRUDiest Movies Database gets all the movies and syncs to $scope. This is the slowest step but happens without the user's awareness.

Users can order the movies by date added, rating, and year. The "Order By" buttons have ng-click set up with two values, order and reverse. These values are passed to the $scope and then to ng-repeat.

The app is responsive. The number of movies displayed per row changes from five in the large view, to four in the medium view, three in the small view, and two on mobile devices.

Clicking on a movie routes the user to the SHOW/EDIT page.

The repository is on GitHub. I also wrote tutorials to make the Firebase app, the Firebase auth, and the asynchronous typeahead.
